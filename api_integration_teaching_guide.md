## 2) ปรัชญาการสอน (อ่านก่อนเป็นอันดับแรก)

โมดูลนี้ออกแบบมาให้ **คนที่ไม่ใช่ผู้เชี่ยวชาญ backend สอนได้** ยึด 3 หลักนี้:

1. **สอน "ความเข้าใจ" ไม่ใช่ "ความเชี่ยวชาญ"** — เป้าหมายคือให้ผู้เรียนอ่านแผนที่ออก ไม่ใช่สร้างถนนเอง คุณไม่ต้องเขียน backend เป็นถึงจะสอนเรื่องนี้ได้
2. **ซื่อสัตย์ = น่าเชื่อถือ** — บอกผู้เรียนตรง ๆ ว่าคุณไม่ใช่สาย backend เต็มตัว แต่จะให้ภาพรวมที่ใช้สั่งงาน AI/dev ได้ ความซื่อสัตย์สร้างความน่าเชื่อถือมากกว่าการแกล้งรู้
3. **ไม่รู้ก็ถาม AI ให้ดูเป็นตัวอย่าง** — พอเจอคำถามลึก ให้เปิด Claude ถามสดหน้าห้อง นี่คือทักษะที่ผู้เรียนจะใช้จริง และ on-brand กับคอร์ส Vibe Code

> **อุปมาหลักที่ใช้ทั้งโมดูล: API = พนักงานเสิร์ฟในร้านอาหาร**
> ลูกค้า(เว็บ) → สั่งผ่านพนักงานเสิร์ฟ(API) → ครัว(back-end/database) → เสิร์ฟกลับมา(response)
> ใช้อุปมานี้ซ้ำทุกครั้งที่เริ่มงง มันครอบคลุมเกือบทุกเรื่องในโมดูล

## 5) การ์ด Status Code (จำให้ขึ้นใจ — ใช้ตอบสดได้ทันที)

| กลุ่ม | แปลว่า | จำง่าย ๆ |
|------|--------|----------|
| **2xx** ✅ | สำเร็จ | 200 OK = ได้ของแล้ว · 201 Created = สร้างใหม่สำเร็จ |
| **4xx** ⚠️ | **เราสั่งผิดเอง** | 400 ส่งมั่ว · 401 ไม่มีบัตรผ่าน · 403 ห้ามเข้า · 404 ไม่เจอ |
| **5xx** 🔥 | **เซิร์ฟเวอร์พัง** | 500 ครัวมีปัญหา (ไม่ใช่ความผิดเรา) |

**ประโยคทอง:** "4xx โทษตัวเอง, 5xx โทษเซิร์ฟเวอร์" — พูดประโยคนี้ซ้ำ ๆ ผู้เรียนจะวิเคราะห์ปัญหาเองได้

## 6) สคริปต์ + จุดเน้นรายบท

### บท 1 — API คืออะไร
- **Key message:** API = พนักงานเสิร์ฟ เราไม่ต้องรู้ว่าครัวทำยังไง แค่รู้ว่าสั่งยังไงและได้อะไรกลับ
- **พูดเปิด:** "เว็บที่ทุกคนเพิ่งทำในคอร์ส Vibe Code ส่วนใหญ่ยัง 'อยู่คนเดียว' วันนี้เราจะทำให้มันคุยกับโลกภายนอกได้ — เครื่องมือที่ทำให้คุยได้เรียกว่า API"
- **อุปมาให้เห็นภาพ:** ถามห้องว่า "เวลาไปร้านอาหาร เราเดินเข้าครัวไปทำเองไหม?" → ไม่ → "เราสั่งผ่านพนักงานเสิร์ฟ API ก็คือพนักงานเสิร์ฟคนนั้น"
- **จุดที่ผู้เรียนมักงง:** คิดว่า API = ลิงก์ URL อันเดียว → ย้ำว่า API คือ "ข้อตกลง" (เมนู) ว่าสั่งอะไรได้และได้อะไรกลับ

### บท 2 — การเดินทางของข้อมูล
- **Key message:** ข้อมูลเดินผ่านหลายชั้น (หน้าเว็บ → API → ครัว → คลัง → ระบบอื่น) และ API คือด่านกลางด่านเดียว
- **อุปมา:** "ถ้าลูกค้าทุกคนเดินเข้าครัวหยิบของเองได้ ครัวจะวุ่นและไม่ปลอดภัย — API คือกฎที่บังคับให้ทุกคนสั่งผ่านพนักงานเสิร์ฟ"
- **เชื่อมงานจริง:** ชี้ตัวอย่างรอบตัวในบท (Login with Google, แผนที่, จ่ายเงิน, อากาศ) ถามห้องว่าเคยเห็นปุ่ม "Login with Google" ไหม — นั่นแหละ API

### บท 3 — กายวิภาค request/response
- **Key message:** 1 คำขอมี 4 อย่าง (Method, URL, Headers, Body) + อ่าน status code เป็น
- **จุดที่ผู้เรียนมักงง:** JSON ดูน่ากลัว → บอกว่ามันแค่ "ชื่อ: ค่า" อ่านออกด้วยตาเปล่า ชี้ตัวอย่างเมนูในบท
- **ย้ำ:** GET ไม่มี Body (แค่ขอดู), POST มี Body (ต้องบอกว่าจะสั่งอะไร) — เดี๋ยวเห็นจริงในบท 4

### บท 4 — ยิง Postman จริง
- ดู **ส่วนที่ 7** เต็ม ๆ (นี่คือช่วงสำคัญสุด)

### บท 5 — ใช้จริง + ถาม AI
- **Key message:** API Key = บัตรเครดิต ห้ามหลุด ห้าม push ขึ้น GitHub
- **ปิดคลาส:** ใช้ประโยคในส่วนที่ 11

## 7) คู่มือ Postman ละเอียด (บทพูด + ขั้นตอน + ผลที่ต้องได้)

> **BASE URL ที่ใช้จริง:** `https://guidebook.bestsolutionskill.com`
> ให้ผู้เรียนพิมพ์ตามนี้ได้เลย (API ทำงานบนเว็บคอร์สเดียวกัน)

### เดโม่ 1 — GET: ขอดูเมนู (เริ่มจากอันนี้เสมอ)
1. เปิด Postman → กดปุ่ม **`+`** เปิดแท็บใหม่
2. Method = **GET**
3. URL = `https://guidebook.bestsolutionskill.com/api/menu`
4. กด **Send**
5. **ผลที่ต้องได้:** Status `200 OK` + JSON เมนู 6 จาน

**พูดหน้าห้อง:** "เราเพิ่งถามร้านว่า ขอดูเมนูหน่อย แล้วร้านส่งรายการอาหารกลับมา — นี่คือ API ทำงาน โดยเราไม่ได้เข้าครัวเลย"

**ต่อยอด:**
- เปลี่ยนเป็น `/api/menu/2` → ได้ต้มยำกุ้งจานเดียว
- เปลี่ยนเป็น `/api/menu/99` → **404** "เห็นไหม ขอของที่ไม่มี ร้านตอบ 404 = 4xx โทษตัวเอง"

### เดโม่ 2 — POST: สั่งอาหาร (เหมือนกด submit ฟอร์ม)
1. Method = **POST**
2. URL = `https://guidebook.bestsolutionskill.com/api/orders`
3. แท็บ **Body** → เลือก **raw** → dropdown ขวาเปลี่ยนเป็น **JSON** ⚠️ (ขั้นนี้คนลืมบ่อยสุด)
4. วาง Body:
```json
{
  "table": 5,
  "items": [
    { "id": 1, "qty": 2 },
    { "id": 6, "qty": 1 }
  ]
}
```
5. กด **Send** → **ผลที่ต้องได้:** Status `201 Created` + `orderId` + `total`

**พูดหน้าห้อง:** "นี่แหละเวลาเว็บกดส่งฟอร์ม — มันส่งข้อมูลเข้าไปแบบนี้ แล้วระบบตอบ 201 ว่าสร้างออเดอร์ให้แล้ว"

**ทำให้พังเพื่อเห็น 400:**
- ลบ `items` เหลือ `{ "table": 5 }` → **400** (ไม่ได้สั่งอะไร)
- สั่ง `{ "items": [{ "id": 5, "qty": 1 }] }` → **400** (ส้มตำของหมด)

### เดโม่ 3 — API Key: เข้าครัว
1. Method = **GET** → URL = `https://guidebook.bestsolutionskill.com/api/kitchen`
2. กด Send (ยังไม่ใส่บัตร) → **401 Unauthorized**
3. แท็บ **Headers** → เพิ่มแถว: Key = `x-api-key`  Value = `best-skills-2026`
4. กด Send อีกครั้ง → **200 OK** + สูตรลับของร้าน

**พูดหน้าห้อง:** "ไม่มีบัตร = 401 นี่คือหน้าตาของระบบที่ต้องเข้าสู่ระบบ หรือใช้ API Key ในงานจริง"

## 8) Playbook เมื่อ "พังหน้าห้อง" (ไม่ต้องตื่นตระหนก)

POST แล้วได้ **400** ทั้งที่ตั้งใจถูก — ไล่เช็ก 4 ข้อนี้ตามลำดับ:

1. **ลืมเลือก Body เป็น raw + JSON** ← เจอบ่อยสุด ส่งไปเป็น text เปล่า
2. **JSON พิมพ์ผิด** — ลืมลูกน้ำ / ใส่ลูกน้ำเกินตัวสุดท้าย / ใช้ `'` แทน `"` (JSON ต้องใช้ double quote `"` เท่านั้น)
3. **เลือก Method ผิด** เผลอเป็น GET
4. **URL พิมพ์ตก** เช่น `order` แทน `orders`

**ถ้ายังแก้ไม่ได้ → เปิด Claude ถามสด (ท่านี้คือการสอน ไม่ใช่จนมุม):**
> "ผมยิง POST ไป /api/orders ด้วย body นี้ [วาง JSON] แล้วได้ status 400 มันแปลว่าอะไร แก้ยังไง"

แล้วพูดกับห้องว่า "เห็นไหม ไม่รู้ก็ถาม AI ได้ นี่คือทักษะจริงที่ทุกคนจะใช้ต่อ"

## 10) FAQ — คำถามที่ผู้เรียนชอบถาม + คำตอบสำหรับผู้สอน

**Q: API กับเว็บไซต์ต่างกันยังไง?**
A: เว็บไซต์คือหน้าจอให้คนดู API คือช่องทางให้ "โปรแกรมคุยกับโปรแกรม" เปิด `/api/menu` จะเห็นแค่ข้อมูลดิบ (JSON) ไม่มีหน้าตาสวย เพราะมันมีไว้ให้เครื่องอ่าน ไม่ใช่คนอ่าน

**Q: ต้องเขียนโค้ดเป็นไหมถึงจะใช้ API ได้?**
A: ไม่ ถ้าใช้ AI ช่วย คุณแค่ต้องเข้าใจคอนเซ็ปต์และสั่งงานให้ถูก — นั่นคือเป้าหมายของวันนี้

**Q: jsonplaceholder / Best Bites มาจากไหน เราทำเองหรือ?**
A: Best Bites เราทำเอง ฝังอยู่ในเว็บคอร์สนี้เลย (API จริง) ส่วน jsonplaceholder เป็นบริการฟรีสาธารณะไว้ฝึก ทั้งคู่ของจริง ใช้ได้

**Q: ทำไมยิง POST แล้วได้ 400?**
A: ดูส่วนที่ 8 — 90% คือลืมเลือก raw+JSON หรือ JSON พิมพ์ผิด

**Q: API Key เอามาจากไหน?**
A: ผู้ให้บริการ API นั้น ๆ ออกให้ตอนเราสมัครใช้ (เช่นสมัคร Google Cloud จะได้ key ของ Google) ในคลาสนี้เราใช้ key จำลอง `best-skills-2026`

**Q: ปลอดภัยไหมถ้าใส่ API Key?**
A: ตัว key ต้องเก็บเป็นความลับ ห้ามแปะในโค้ดหน้าเว็บ ห้าม push ขึ้น GitHub — เหมือนรหัสบัตรเครดิต (ดูบท 5.1)

**Q: ของที่สั่งไปเก็บที่ไหน ทำไมสั่งซ้ำได้เรื่อย ๆ?**
A: ร้าน Best Bites เป็นร้านจำลองเพื่อฝึก ไม่ได้เก็บจริง จึงยิงกี่ครั้งก็ได้ ไม่พังข้อมูลใคร

**เปิด:**
> "ช่วงนี้ผมไม่ได้จะสอนให้คุณเขียน backend นะครับ ตัวผมเองก็ไม่ใช่สาย backend เต็มตัว แต่สิ่งที่ผมจะให้คือ 'แผนที่' ให้เข้าใจว่าเว็บคุยกับระบบอื่นยังไง พอเข้าใจภาพนี้ เวลาสั่ง AI หรือคุยกับโปรแกรมเมอร์ คุณจะรู้เรื่องและสั่งงานเป็น วันนี้เราจะลองยิง API จริงด้วยกันใน Postman คนละไม่กี่คลิก"

**ปิด:**
> "จบวันนี้คุณไม่ได้เป็น backend developer และไม่จำเป็นต้องเป็น สิ่งที่ได้คือแผนที่ที่ทำให้ไม่หลงทาง และคุยกับ AI/dev รู้เรื่อง จำไว้ว่า 'ไม่รู้' ไม่ใช่เรื่องน่าอาย แต่ 'ไม่รู้แล้วถาม AI เป็น' คือทักษะจริงของยุคนี้"
