# 01 - Vibe Code Foundation

## บทนี้เรียนไปเพื่ออะไร

ก่อนเริ่มสร้างเว็บ ผู้เรียนต้องเข้าใจก่อนว่า AI ไม่ได้แทนการคิดโจทย์ของมนุษย์ AI ช่วยให้ทำงานเร็วขึ้น เขียนโค้ดเร็วขึ้น และแก้ปัญหาได้เป็นระบบขึ้น แต่ AI จะให้ผลลัพธ์ดีแค่ไหนขึ้นอยู่กับความชัดเจนของโจทย์ที่เราป้อนเข้าไป

บทนี้จึงเน้นการทำให้ผู้เรียนเปลี่ยนไอเดียที่ยังคลุมเครือให้กลายเป็น Project Brief ที่ AI เข้าใจ และมนุษย์ตรวจทานได้

## Vibe Code คืออะไร

Vibe Code คือแนวทางการทำงานที่ใช้ AI ช่วยสร้าง แก้ไข และพัฒนาเว็บไซต์ โดยผู้เรียนเป็นคนกำหนดทิศทาง และให้ AI ช่วยทำงานในส่วนที่ใช้เวลาเยอะ เช่น เขียนโค้ด สร้าง Layout แก้ Error ปรับข้อความ หรือเสนอทางเลือกด้าน UX/UI

สิ่งที่ต้องย้ำ:

- AI ไม่ได้แทนการคิดโจทย์
- AI ไม่ได้รู้เป้าหมายธุรกิจเอง
- ถ้า Brief ไม่ชัด เว็บที่ได้จะไม่ตรงจุด
- คนสั่ง AI ต้องรู้ว่ากำลังต้องการผลลัพธ์อะไร
- การทำเว็บที่ดีต้องเริ่มจาก Strategy ก่อน Design และ Development

ประโยคสำคัญ:

> AI ช่วยทำเว็บให้เร็วขึ้น แต่ถ้าเราไม่รู้ว่าเว็บต้องแก้ปัญหาอะไร AI ก็สร้างได้แค่เว็บที่ดูสวย แต่ไม่ตอบโจทย์

## AI ช่วยทำเว็บไซต์ตรงไหนได้บ้าง

### 1. ช่วยคิดไอเดียเว็บไซต์

AI ช่วยแตกไอเดียจากโจทย์กว้าง ๆ ให้เป็นประเภทเว็บไซต์ที่ทำได้จริง เช่น Landing Page ขายบริการ, Portfolio, เว็บร้านอาหาร, เว็บคลินิก, เว็บรับ Lead หรือเว็บขายคอร์ส

ตัวอย่าง prompt:

```text
ฉันอยากทำเว็บไซต์สำหรับธุรกิจรับทำเว็บไซต์ให้ SME
ช่วยเสนอไอเดียเว็บไซต์ 5 แบบ พร้อมเป้าหมาย กลุ่มเป้าหมาย และ CTA หลักของแต่ละแบบ
```

### 2. ช่วยเขียน Project Brief

AI ช่วยจัดข้อมูลกระจัดกระจายให้เป็น Brief ที่อ่านง่าย เช่น เป้าหมายเว็บไซต์ กลุ่มเป้าหมาย Pain Point จุดขาย Offer CTA Section และ Mood & Tone

### 3. ช่วยวาง Section และ User Flow

AI ช่วยเรียงลำดับการเล่าเรื่องบนหน้าเว็บ เช่น Hero, Problem, Solution, Services, Process, Portfolio, Pricing, FAQ และ Contact

### 4. ช่วยเขียน Copywriting

AI ช่วยเขียน Headline, Subheadline, CTA, FAQ และข้อความขายในแต่ละ Section ได้หลายเวอร์ชัน เพื่อให้ผู้เรียนเลือกและปรับให้เข้ากับธุรกิจจริง

### 5. ช่วยสร้าง HTML Preview

AI ช่วยสร้างหน้าเว็บตัวอย่างด้วย HTML และ CSS เพื่อให้เห็นภาพดีไซน์ก่อนเข้าสู่โปรเจกต์จริง

### 6. ช่วยแปลงเป็น Next.js

AI ช่วยแปลง HTML เป็น Component และสร้างโครงสร้างโปรเจกต์จริง เช่น `app/page.tsx`, `components/Hero.tsx`, `components/Pricing.tsx`

### 7. ช่วย Debug

AI ช่วยอ่าน Error อธิบายสาเหตุ และแนะนำวิธีแก้ทีละขั้นตอน แต่ผู้เรียนต้องส่ง context ให้ครบ เช่น คำสั่งที่รัน Error ที่ขึ้น และไฟล์ที่เพิ่งแก้

## การเลือกโจทย์เว็บไซต์

โจทย์ที่ดีสำหรับคอร์สนี้ควรมีลักษณะดังนี้:

- ทำจบเป็นหน้าเว็บหลักได้ภายใน 2 วัน
- มีกลุ่มเป้าหมายชัดเจน
- มี CTA ที่วัดผลได้
- ไม่ต้องมีระบบซับซ้อนตั้งแต่ต้น
- สามารถใช้เป็น Portfolio หรือใช้งานจริงต่อได้

โจทย์ที่ควรหลีกเลี่ยงในคลาสเริ่มต้น:

- Marketplace หลายผู้ขาย
- E-commerce เต็มระบบพร้อมตะกร้าและจ่ายเงิน
- ระบบสมาชิกซับซ้อน
- Dashboard หลาย Role
- Social Network
- App ที่ต้องใช้ Realtime หรือ Chat เต็มรูปแบบ

เหตุผลคือคอร์สนี้ต้องการให้ผู้เรียนเข้าใจ Workflow การสร้างเว็บให้ครบก่อน ไม่ใช่ติดอยู่กับระบบใหญ่เกินเวลาคลาส

## เปลี่ยนไอเดียให้เป็น Project Brief

Project Brief คือเอกสารสั้น ๆ ที่บอกว่าเว็บนี้คืออะไร ทำเพื่อใคร แก้ปัญหาอะไร มีจุดขายอะไร และต้องการให้ผู้ใช้ทำอะไร

Brief ที่ดีช่วยให้:

- ผู้เรียนคิดโจทย์ชัดขึ้น
- AI สร้างผลลัพธ์ตรงขึ้น
- แก้งานง่ายขึ้น เพราะมีเกณฑ์ตัดสิน
- Design และ Copy ไม่หลุดจากเป้าหมาย
- ใช้เป็นเอกสารตั้งต้นสำหรับทำเว็บจริง

## โครงสร้าง Project Brief

Project Brief ควรมีหัวข้อเหล่านี้:

```text
1. ชื่อโปรเจกต์
2. ประเภทเว็บไซต์
3. เป้าหมายของเว็บไซต์
4. กลุ่มเป้าหมาย
5. Pain Point ของกลุ่มเป้าหมาย
6. จุดขายหลัก
7. Offer หรือสิ่งที่นำเสนอ
8. CTA หลัก
9. CTA รอง
10. Tone & Mood
11. สีและสไตล์ที่ต้องการ
12. Section ที่คาดว่าจะมี
13. ข้อมูลที่ต้องการให้ลูกค้าส่งมา
14. ตัวอย่างเว็บไซต์อ้างอิง
15. สิ่งที่ไม่ต้องการให้มีในเว็บ
16. ผลลัพธ์สุดท้ายที่ต้องการ
```

## ตัวอย่าง Project Brief แบบย่อ

```md
# Project Brief: SME Website Starter

## ประเภทเว็บไซต์
Landing Page ขายบริการรับทำเว็บไซต์

## เป้าหมายของเว็บไซต์
ทำให้เจ้าของธุรกิจ SME เห็นว่าเว็บไซต์ช่วยเพิ่มความน่าเชื่อถือ และชวนให้ทัก LINE เพื่อคุยโจทย์

## กลุ่มเป้าหมาย
เจ้าของธุรกิจ SME ที่มีเพจอยู่แล้ว แต่ยังไม่มีเว็บไซต์หลักของตัวเอง

## Pain Point
- ลูกค้ายังไม่มั่นใจเมื่อเห็นแค่ Facebook Page
- ยิงแอดแล้วไม่มีหน้าเว็บรองรับ
- อยากมีเว็บแต่ไม่รู้จะเริ่มจากข้อมูลอะไร
- กลัวจ้างทำเว็บแล้วคุยไม่รู้เรื่อง

## จุดขาย
ช่วยวางโครงเว็บ ออกแบบหน้าแรก พัฒนาเว็บ และ Deploy ให้พร้อมใช้งาน โดยใช้ภาษาที่เจ้าของธุรกิจเข้าใจ

## CTA หลัก
ทัก LINE เพื่อขอคำแนะนำเว็บไซต์

## Mood & Tone
Modern, Clean, Premium, Friendly
```

## Workshop: เลือกโปรเจกต์และเขียน Project Brief

### สิ่งที่ผู้เรียนต้องทำ

1. เลือกประเภทเว็บไซต์ที่จะทำ
2. เขียนเป้าหมายของเว็บไซต์
3. ระบุกลุ่มเป้าหมาย
4. เขียน Pain Point ของกลุ่มเป้าหมาย
5. เขียนจุดขายหลัก
6. เลือก CTA หลัก
7. ใช้ AI ช่วยจัดเป็น Project Brief
8. ตรวจและปรับ Brief ให้เข้าใจง่าย

### Prompt สำหรับ Workshop

```text
ช่วยสร้าง Project Brief สำหรับเว็บไซต์จากข้อมูลนี้

ประเภทเว็บไซต์:
เป้าหมายของเว็บไซต์:
กลุ่มเป้าหมาย:
ปัญหาของกลุ่มเป้าหมาย:
จุดขาย:
Offer:
Tone & Mood:
สีที่ต้องการ:
CTA หลัก:
ข้อมูลที่ต้องการให้ลูกค้าทำ:

ช่วยจัดเป็น Project Brief แบบละเอียด พร้อมแนะนำ Section ที่ควรมีในหน้าเว็บ
```

### ผลลัพธ์ที่ต้องได้

ผู้เรียนต้องมี Project Brief ของตัวเอง 1 ชุด พร้อมนำไปใช้สร้างโครงหน้าเว็บใน Session ถัดไป

## สรุปท้ายบท

- AI ช่วยทำเว็บได้ดีเมื่อมนุษย์ให้โจทย์ชัด
- Project Brief คือฐานของทั้ง Design, Copy และ Development
- เว็บที่ดีต้องรู้ว่าใครคือผู้ใช้ และอยากให้ผู้ใช้ทำอะไร
- ควรเริ่มจากเว็บขนาดเหมาะสม เช่น Landing Page หรือ Portfolio
- ผลลัพธ์ของ Session นี้คือ Brief ที่พร้อมนำไปวาง Section
