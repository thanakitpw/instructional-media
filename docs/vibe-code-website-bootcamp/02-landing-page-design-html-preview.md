# 02 - Landing Page Design & HTML Preview

## บทนี้เรียนไปเพื่ออะไร

เมื่อมี Project Brief แล้ว ขั้นต่อไปคือเปลี่ยนโจทย์ให้เป็นโครงหน้าเว็บที่ผู้ใช้เข้าใจได้จริง บทนี้สอนให้ผู้เรียนวาง Section, User Flow, CTA, UX/UI Direction และใช้ AI สร้าง HTML Preview เพื่อดูหน้าตาก่อนขึ้นโปรเจกต์ Next.js

เป้าหมายไม่ใช่การทำเว็บให้สวยที่สุดตั้งแต่รอบแรก แต่คือการสร้างต้นแบบที่ชัดพอให้ตรวจ แก้ และนำไปพัฒนาต่อได้

## Sitemap คืออะไร

Sitemap คือโครงสร้างหน้าของเว็บไซต์ เช่น:

```text
Home
About
Services
Portfolio
Contact
```

สำหรับคอร์สนี้ แนะนำให้เริ่มจากเว็บหน้าเดียวก่อน เพราะเหมาะกับเวลาเรียนและทำให้ผู้เรียนเห็นผลลัพธ์เร็ว

ตัวอย่าง Sitemap สำหรับ Landing Page:

```text
Landing Page
Thank You Page
Privacy Policy
```

## User Flow คืออะไร

User Flow คือเส้นทางที่ผู้ใช้เดินผ่านหน้าเว็บ ตั้งแต่เห็นหน้าแรกจนถึงการตัดสินใจทำ Action

ตัวอย่าง Flow:

```text
เห็น Hero
↓
เข้าใจปัญหาของตัวเอง
↓
เห็น Solution
↓
ดูบริการและขั้นตอน
↓
ดูตัวอย่างหรือราคา
↓
อ่าน FAQ เพื่อลดความกังวล
↓
กด CTA เพื่อติดต่อ
```

## Section ที่ควรมีใน Landing Page

### 1. Hero Section

Hero คือส่วนแรกที่ผู้ใช้เห็น ต้องตอบให้ได้ว่า:

- เว็บนี้คืออะไร
- ช่วยใคร
- ช่วยแก้ปัญหาอะไร
- มีจุดขายอะไร
- ต้องกดอะไรต่อ

ตัวอย่าง:

```text
Headline: ทำเว็บไซต์ธุรกิจให้ดูน่าเชื่อถือ พร้อมใช้งานจริง
Subheadline: เหมาะสำหรับเจ้าของธุรกิจที่อยากมีเว็บไซต์ไว้รับลูกค้าใหม่ โดยไม่ต้องเริ่มจากศูนย์
CTA: ทัก LINE เพื่อขอดูแนวทางเว็บไซต์
Secondary CTA: ดูตัวอย่างผลงาน
```

### 2. Problem Section

ใช้เล่าปัญหาของกลุ่มเป้าหมาย เช่น มีเพจแล้วแต่ลูกค้ายังไม่มั่นใจ ยิงแอดแล้วไม่มีหน้าเว็บรองรับ อยากมีเว็บไซต์แต่ไม่รู้จะเริ่มอย่างไร หรือกลัวจ้างทำเว็บแล้วไม่ตรงใจ

หลักการเขียน Problem Section:

- เขียนจากมุมของลูกค้า
- ใช้ภาษาที่ลูกค้าพูดจริง
- ไม่ทำให้ลูกค้ารู้สึกผิด
- เชื่อมไปหา Solution ได้

### 3. Solution Section

ใช้บอกว่าวิธีของเราช่วยแก้ปัญหาอย่างไร เช่น ช่วยวางโครงเว็บ ออกแบบหน้าแรก ทำเว็บให้รองรับมือถือ มีปุ่มติดต่อชัดเจน และพร้อม Deploy ขึ้นออนไลน์

Solution ที่ดีควรตอบปัญหาจาก Section ก่อนหน้า ไม่ใช่พูดแค่ฟีเจอร์ของบริการ

### 4. Service / Feature Section

แสดงสิ่งที่ลูกค้าจะได้รับ เช่น:

- ออกแบบหน้าเว็บ
- พัฒนาเว็บไซต์
- รองรับมือถือ
- เชื่อมฟอร์มติดต่อ
- ติดตั้ง Tracking
- ดูแลหลังส่งมอบ

ควรเขียน Feature ให้โยงกับประโยชน์ เช่น "รองรับมือถือ" ไม่ใช่แค่บอกว่าทำได้ แต่ควรอธิบายว่า "ลูกค้าเปิดจากโทรศัพท์แล้วอ่านง่ายและกดติดต่อได้ทันที"

### 5. Process Section

แสดงขั้นตอนการทำงานเพื่อเพิ่มความมั่นใจ:

```text
1. คุยโจทย์
2. วางโครงเว็บ
3. ออกแบบหน้าแรก
4. พัฒนาเว็บ
5. ตรวจงาน
6. Deploy
7. ส่งมอบ
```

### 6. Portfolio / Example Section

ใช้แสดงตัวอย่างงาน Mockup, Case Study, Before / After หรือ Demo Project ถ้าผู้เรียนยังไม่มีผลงานจริง สามารถใช้ Demo Project เป็นตัวอย่าง Portfolio ได้

### 7. Pricing Section

ใช้แสดงราคา แพ็กเกจ หรือราคาเริ่มต้น เช่น:

```text
Starter: เหมาะกับเริ่มต้น
Growth: เหมาะกับธุรกิจที่ต้องการหน้าเว็บครบขึ้น
Business: เหมาะกับธุรกิจที่ต้องการระบบและการวัดผล
```

ถ้ายังไม่อยากใส่ราคาเต็ม สามารถใช้ "ราคาเริ่มต้น" หรือ "ขอใบเสนอราคา" แทนได้

### 8. FAQ Section

ใช้ตอบข้อกังวล เช่น:

- ใช้เวลากี่วัน
- ต้องเตรียมข้อมูลอะไร
- แก้ไขได้กี่ครั้ง
- มีค่าใช้จ่ายรายปีไหม
- เว็บรองรับมือถือไหม
- หลังส่งมอบแก้ไขเองได้ไหม

FAQ ที่ดีช่วยลดแรงเสียดทานก่อนลูกค้ากด CTA

### 9. Final CTA Section

ใช้ปิดท้ายและชวนให้ผู้ใช้ทำ Action:

```text
อยากมีเว็บไซต์ที่ช่วยให้ธุรกิจดูน่าเชื่อถือขึ้น?
ทัก LINE เพื่อเริ่มคุยโจทย์ได้เลย
```

## พื้นฐาน UX/UI ที่จำเป็น

### Layout

- หัวข้อต้องเด่น
- ข้อความไม่ควรยาวเกินไป
- Section ต้องมีระยะห่าง
- CTA ต้องมองเห็นง่าย
- หน้าเว็บควรอ่านไหลจากบนลงล่าง

### Typography

- ใช้ฟอนต์ไม่เยอะเกินไป
- ขนาดหัวข้อและเนื้อหาต้องต่างกันชัดเจน
- ตัวหนังสือต้องอ่านง่ายบนมือถือ
- ไม่ควรใช้ข้อความแน่นเกินไป

### Color

- มีสีหลัก 1 สี
- มีสีรอง 1-2 สี
- พื้นหลังต้องช่วยให้อ่านง่าย
- สีปุ่ม CTA ต้องเด่นกว่าส่วนอื่น

ตัวอย่างโทนสี:

```text
Primary: Orange
Background: White / Light Gray
Text: Black / Dark Gray
Accent: Soft Orange
```

### Spacing

- ระยะห่างระหว่าง Section ต้องพอดี
- Card ไม่ควรแน่นเกินไป
- ปุ่มควรมีพื้นที่รอบ ๆ
- หน้าเว็บต้องอ่านสบาย ไม่อึดอัด

### Responsive

- หน้าเว็บต้องดูดีบนมือถือ
- Card ควรเรียงลงมาในมือถือ
- ปุ่มต้องกดง่าย
- ข้อความต้องไม่เล็กเกินไป
- รูปภาพต้องไม่ล้นจอ

## Prompt สำหรับวาง Section

```text
จาก Project Brief ด้านล่าง ช่วยวางโครงหน้า Landing Page แบบละเอียด

ต้องการให้ช่วยจัด:
1. Section ทั้งหมดที่ควรมี
2. ลำดับการเล่าเรื่องจากบนลงล่าง
3. เป้าหมายของแต่ละ Section
4. Headline ของแต่ละ Section
5. ข้อความอธิบายสั้น ๆ
6. CTA ที่เหมาะสม
7. คำแนะนำ UX/UI
8. แนวทาง Responsive บนมือถือ

Project Brief:
[วาง Project Brief ตรงนี้]
```

## Prompt สำหรับสร้าง HTML Preview

```text
ช่วยสร้าง HTML Preview สำหรับ Landing Page จากโครงสร้างด้านล่าง

เงื่อนไข:
- ใช้ HTML + CSS ในไฟล์เดียว
- ดีไซน์สไตล์ Modern, Clean, Premium
- ใช้สีหลักเป็นส้ม ขาว ดำ
- Layout ต้องดูน่าเชื่อถือ
- มี Hero, Problem, Solution, Services, Process, Pricing, FAQ และ Contact CTA
- รองรับ Responsive เบื้องต้น
- ใช้ข้อความภาษาไทย
- ปุ่ม CTA เด่นชัด
- ไม่ต้องเชื่อมระบบจริง แค่ทำเป็น Preview

ข้อมูลหน้าเว็บ:
[วาง Section ที่ได้จาก AI ตรงนี้]
```

## วิธีตรวจ HTML Preview

ตรวจด้วยคำถามเหล่านี้:

- Hero เข้าใจใน 5 วินาทีหรือไม่
- Headline ชัดเจนหรือไม่
- CTA เด่นพอหรือไม่
- Section เรียงลำดับดีหรือไม่
- สีอ่านง่ายหรือไม่
- ฟอนต์ดูน่าเชื่อถือหรือไม่
- ข้อความขายยาวเกินไปหรือไม่
- มีข้อมูลช่วยให้ลูกค้าตัดสินใจพอหรือไม่
- มี FAQ ตอบข้อกังวลหรือไม่
- ปุ่มติดต่ออยู่ในตำแหน่งที่เหมาะสมหรือไม่
- เปิดบนมือถือแล้วใช้งานได้หรือไม่
- มี Placeholder ที่ยังไม่ได้แก้หรือไม่

## Workshop: สร้าง HTML Preview ของเว็บไซต์ตัวเอง

### สิ่งที่ผู้เรียนต้องทำ

1. นำ Project Brief จากช่วงเช้ามาใช้
2. ให้ AI ช่วยวาง Section
3. ปรับ Section ให้เหมาะกับเว็บของตัวเอง
4. ให้ AI สร้าง HTML Preview
5. เปิดไฟล์ Preview ดูใน Browser
6. ปรับข้อความ สี Layout และ CTA
7. ตรวจด้วย Checklist
8. Save ไฟล์ HTML Preview ไว้ใช้ต่อ Day 2

### ผลลัพธ์ที่ต้องได้

ผู้เรียนต้องมี HTML Preview ของเว็บตัวเอง 1 หน้า ที่เห็นภาพดีไซน์ชัดเจน พร้อมนำไปแปลงเป็น Next.js

## สรุปท้ายบท

- Landing Page ต้องเล่าเรื่องจากปัญหาไปสู่ Action
- Section แต่ละส่วนควรมีหน้าที่ชัดเจน
- HTML Preview คือพื้นที่ทดลองก่อนเข้าสู่โค้ดจริง
- UX/UI พื้นฐานที่สำคัญคือความชัดเจน อ่านง่าย และกด CTA ได้
- ก่อนจบ Day 1 ผู้เรียนต้องมี Preview ที่ตรวจแล้วว่านำไปต่อได้
