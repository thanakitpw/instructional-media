# 08 - Workshop Checklists

ไฟล์นี้ใช้ตรวจงานของผู้เรียนในแต่ละช่วงของคอร์ส

## Day 1 Morning: Project Brief Checklist

### Strategy Checklist

- รู้ว่าเว็บทำเพื่ออะไร
- รู้ว่ากลุ่มเป้าหมายคือใคร
- รู้ว่า Pain Point หลักคืออะไร
- มีจุดขายชัดเจน
- มี Offer ที่อธิบายได้
- มี CTA หลัก
- มี CTA รอง ถ้าจำเป็น
- มี Mood & Tone
- มีสีหรือสไตล์ที่ต้องการ
- มีรายการข้อมูลที่ต้องเตรียม

### Brief Quality Checklist

- อ่านแล้วเข้าใจใน 1-2 นาที
- ไม่กว้างเกินไป
- ไม่ใช้คำสวยแต่ไม่ชัด เช่น "ทันสมัย" โดยไม่อธิบายเพิ่ม
- ระบุกลุ่มเป้าหมายเฉพาะเจาะจง
- บอกผลลัพธ์ที่ต้องการจากเว็บได้
- ใช้เป็น input ให้ AI วาง Section ต่อได้

## Day 1 Afternoon: HTML Preview Checklist

### Content Checklist

- Hero Section เข้าใจใน 5 วินาที
- Headline ชัดเจน
- Subheadline อธิบายประโยชน์ได้
- CTA หลักเด่น
- Problem Section ตรงกับกลุ่มเป้าหมาย
- Solution เชื่อมกับ Problem
- Services หรือ Features ชัดเจน
- Process ทำให้เชื่อมั่นขึ้น
- Pricing หรือ Offer ไม่สับสน
- FAQ ตอบข้อกังวลสำคัญ
- Final CTA ชวนให้ตัดสินใจ

### Design Checklist

- สีอ่านง่าย
- ฟอนต์อ่านง่าย
- Section มีระยะห่างพอดี
- Card ไม่แน่นเกินไป
- CTA มองเห็นง่าย
- หน้าเว็บไม่ดูรก
- ไม่มีสีที่แข่งกันมากเกินไป
- ภาพหรือ icon ไม่ทำให้เนื้อหาสับสน

### Responsive Checklist

- เปิดบนมือถือแล้วไม่ล้นจอ
- ข้อความไม่เล็กเกินไป
- ปุ่มกดง่าย
- Card เรียงลงมาเหมาะสม
- Navbar หรือเมนูไม่บังเนื้อหา
- รูปไม่แตกหรือบิด

### Cleanup Checklist

- ไม่มี Placeholder เช่น Lorem ipsum
- ไม่มีลิงก์หลอกที่ควรแก้แล้ว
- ข้อมูลติดต่อถูกต้อง
- ชื่อแบรนด์ถูกต้อง
- CTA ใช้คำเดียวกันอย่างสม่ำเสมอ

## Day 2 Morning: Next.js Build Checklist

### Project Setup

- สร้างโปรเจกต์ Next.js แล้ว
- ใช้ TypeScript
- ใช้ Tailwind CSS
- รัน `npm run dev` ได้
- เปิดเว็บในเครื่องได้
- โครงไฟล์อ่านง่าย

### Component Checklist

- แยก Component หลักแล้ว
- มี `Navbar` ถ้าจำเป็น
- มี `Hero`
- มี `Problem`
- มี `Solution`
- มี `Services`
- มี `Process`
- มี `Pricing` หรือ Offer
- มี `FAQ`
- มี `ContactCTA` หรือ Form
- มี `Footer`

### Code Quality Checklist

- ไม่มี Component ใหญ่เกินไปโดยไม่จำเป็น
- ชื่อไฟล์เข้าใจง่าย
- ข้อความสำคัญแก้ได้ง่าย
- className ไม่ซ้ำซ้อนเกินไป
- ไม่มี unused import
- ไม่มี console error สำคัญ
- Build ผ่าน หรือรู้ Error ที่ต้องแก้

### GitHub Checklist

- ใช้ Git แล้ว
- Commit แรกเรียบร้อย
- มี GitHub Repository
- Push โค้ดขึ้น GitHub แล้ว
- README มีคำสั่งรันเบื้องต้น

## Day 2 Afternoon: Deploy & Tracking Checklist

### Deploy Checklist

- Push โค้ดล่าสุดขึ้น GitHub แล้ว
- Import Project เข้า Vercel แล้ว
- Framework เป็น Next.js
- ตั้ง Environment Variables ถ้ามี
- Deploy ผ่าน
- เปิด URL จริงได้
- หน้าเว็บไม่พังบน Production

### Post-Deploy Checklist

- รูปโหลดครบ
- ปุ่ม CTA ใช้งานได้
- Link LINE / Phone / Email ถูกต้อง
- Form ส่งข้อมูลได้ หรือมีสถานะจำลองชัดเจน
- เปิดบนมือถือได้
- Title เว็บถูกต้อง
- Description เว็บถูกต้อง
- ไม่มีข้อมูลทดสอบที่ไม่ควรโชว์

### Tracking Checklist

- รู้ว่า GA4 ใช้ทำอะไร
- รู้ว่า Google Tag ใช้ทำอะไร
- รู้ว่า Search Console ใช้ทำอะไร
- ระบุ Conversion หลักแล้ว
- ตั้งชื่อ Key Event แล้ว
- รู้ว่าจะวัด click, submit หรือ page view อะไร
- มีแผนตรวจผลหลังเว็บออนไลน์

### Performance Checklist

- รูปไม่ใหญ่เกินไป
- ใช้ภาพ WebP ถ้าเป็นไปได้
- ไม่มี animation หนักเกินไป
- ตรวจ PageSpeed Insights แล้ว
- ตรวจ Accessibility เบื้องต้นแล้ว
- ใส่ Alt Text ให้รูปสำคัญ

## Final Course Checklist

ผู้เรียนควรมี:

- Project Brief
- Landing Page Section Plan
- HTML Preview
- Next.js Project
- Component หลัก
- GitHub Repository
- Vercel URL
- Lead Form หรือ Contact Form
- Tracking Plan
- Portfolio / Business Roadmap

## Instructor Quick Review

ใช้คำถามเหล่านี้ตรวจผลงานท้ายคอร์ส:

1. เว็บนี้ทำเพื่อใคร
2. ผู้ใช้ควรกดอะไร
3. Hero เข้าใจเร็วไหม
4. Section เรียงเป็นเหตุเป็นผลไหม
5. CTA เด่นและกดได้จริงไหม
6. เปิดบนมือถือแล้วใช้งานได้ไหม
7. โค้ดแยก Component พอแก้ต่อได้ไหม
8. Deploy แล้วเปิด URL ได้ไหม
9. ผู้เรียนอธิบายสิ่งที่ทำเองได้ไหม
10. เว็บนี้นำไปใช้ต่อเป็น Portfolio หรือธุรกิจได้ไหม
