# 01 - เส้นทางการเรียน

## ภาพรวม

คอร์สนี้แบ่งเป็น 2 วัน แต่ละช่วงจะมีงานที่ต้องทำให้เสร็จ เพื่อให้ท้ายคอร์สคุณมีเว็บไซต์จริงที่เปิดออนไลน์ได้

แนวคิดสำคัญ:

> เริ่มจากโจทย์ให้ชัดก่อน แล้วค่อยใช้ AI ช่วยออกแบบ เขียนโค้ด แก้บั๊ก และนำเว็บขึ้นใช้งานจริง

## Day 1: จากไอเดียสู่ HTML Preview

### ช่วงที่ 1: เลือกโจทย์และเขียน Project Brief

คุณจะเลือกว่าจะทำเว็บประเภทไหน เช่น:

- Landing Page ขายบริการ
- Portfolio Website
- เว็บไซต์ธุรกิจ
- เว็บเก็บ Lead หรือฟอร์มติดต่อ

สิ่งที่ต้องทำ:

1. เลือกประเภทเว็บไซต์
2. ระบุกลุ่มเป้าหมาย
3. เขียนปัญหาของกลุ่มเป้าหมาย
4. เขียนจุดขายหรือข้อเสนอของคุณ
5. เลือก CTA หลัก เช่น ทัก LINE, ส่งฟอร์ม, โทร, สมัครเรียน
6. ใช้ AI ช่วยจัดข้อมูลเป็น Project Brief

ผลลัพธ์ที่ต้องได้:

- Project Brief 1 ชุด

### ช่วงที่ 2: วางโครงหน้าเว็บและทำ HTML Preview

คุณจะนำ Project Brief มาแปลงเป็นโครงหน้าเว็บ

Section หลักที่มักใช้:

- Hero
- Problem
- Solution
- Services / Features
- Process
- Portfolio / Example
- Pricing / Offer
- FAQ
- Final CTA

สิ่งที่ต้องทำ:

1. ใช้ AI ช่วยวาง Section
2. ตรวจว่า Section เรียงเป็นเรื่องเดียวกันหรือไม่
3. ใช้ AI สร้าง HTML Preview
4. เปิด Preview ใน Browser
5. ปรับข้อความ สี ปุ่ม และ Layout
6. ตรวจบนมือถือ

ผลลัพธ์ที่ต้องได้:

- HTML Preview ของเว็บไซต์ตัวเอง 1 หน้า

## Day 2: จาก HTML Preview สู่เว็บไซต์ออนไลน์

### ช่วงที่ 3: แปลงเป็น Next.js

คุณจะเปลี่ยน HTML Preview ให้เป็นโปรเจกต์ Next.js ที่แก้ต่อได้จริง

สิ่งที่ต้องทำ:

1. สร้างโปรเจกต์ Next.js
2. ให้ AI ช่วยแปลง HTML เป็น Component
3. แยกส่วนหน้าเว็บ เช่น Hero, Pricing, FAQ, Footer
4. รันเว็บในเครื่อง
5. แก้ Error เบื้องต้น
6. Commit และ Push ขึ้น GitHub

ผลลัพธ์ที่ต้องได้:

- Next.js Project ที่รันได้
- GitHub Repository

### ช่วงที่ 4: Deploy และวางแผนวัดผล

คุณจะนำเว็บไซต์ขึ้นออนไลน์ และคิดว่าเว็บนี้ควรวัดผลอะไร

สิ่งที่ต้องทำ:

1. ตรวจเว็บก่อน Deploy
2. Deploy ผ่าน Vercel
3. เปิด URL จริง
4. ตรวจปุ่มและฟอร์ม
5. วางแผน Conversion / Key Event
6. เขียน Roadmap ว่าจะพัฒนาต่ออย่างไร

ผลลัพธ์ที่ต้องได้:

- Vercel URL
- Checklist หลัง Deploy
- Tracking Plan
- Roadmap หลังเรียน

## สิ่งที่ควรจำระหว่างเรียน

- อย่าเริ่มจากโค้ดถ้าโจทย์ยังไม่ชัด
- อย่าให้ AI แก้ทั้งโปรเจกต์ถ้าต้องแก้แค่ส่วนเดียว
- ส่งข้อมูลให้ AI แบบเฉพาะเจาะจง
- อ่านผลลัพธ์ของ AI ทุกครั้งก่อนนำไปใช้
- เว็บที่ดีไม่ใช่แค่สวย แต่ต้องทำให้ผู้ใช้เข้าใจและตัดสินใจได้
