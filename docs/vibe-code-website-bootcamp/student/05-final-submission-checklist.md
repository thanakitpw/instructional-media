# 05 - Final Submission Checklist

ใช้ไฟล์นี้ตรวจงานก่อนส่งท้ายคอร์ส

## 1. Strategy

- [ ] ฉันอธิบายได้ว่าเว็บนี้ทำเพื่ออะไร
- [ ] ฉันรู้ว่าใครคือกลุ่มเป้าหมาย
- [ ] ฉันรู้ว่าเว็บนี้แก้ปัญหาอะไร
- [ ] ฉันมี CTA หลักที่ชัดเจน
- [ ] ฉันมี Project Brief

## 2. Content & Design

- [ ] Hero Section เข้าใจง่าย
- [ ] Headline ชัด
- [ ] CTA เด่น
- [ ] Section เรียงจากปัญหาไปสู่การตัดสินใจ
- [ ] มี FAQ หรือข้อมูลตอบข้อกังวล
- [ ] สีและฟอนต์อ่านง่าย
- [ ] เปิดบนมือถือแล้วไม่ล้นจอ
- [ ] ไม่มี Placeholder

## 3. HTML Preview

- [ ] มี HTML Preview
- [ ] เปิดใน Browser ได้
- [ ] ตรวจ Responsive แล้ว
- [ ] จดสิ่งที่ต้องแก้ก่อนขึ้น Next.js แล้ว

## 4. Next.js Project

- [ ] มีโปรเจกต์ Next.js
- [ ] รัน `npm run dev` ได้
- [ ] แยก Component หลักแล้ว
- [ ] ไม่มี Error สำคัญใน Terminal
- [ ] ไม่มี unused หรือ code ที่ทำให้เว็บพัง
- [ ] หน้าเว็บยังตรงกับ Preview หลัก

## 5. GitHub

- [ ] Commit งานแล้ว
- [ ] Push ขึ้น GitHub แล้ว
- [ ] เปิด Repository ได้
- [ ] มี README หรือคำอธิบายเบื้องต้น

## 6. Deploy

- [ ] Deploy ผ่าน Vercel แล้ว
- [ ] เปิด Vercel URL ได้
- [ ] ปุ่ม CTA ใช้งานได้
- [ ] รูปโหลดครบ
- [ ] ฟอร์มใช้งานได้ หรือมีแผนต่อยอดชัดเจน
- [ ] เปิดบนมือถือได้

## 7. Tracking Plan

- [ ] ระบุ Conversion หลักแล้ว
- [ ] ตั้งชื่อ Key Event แล้ว
- [ ] รู้ว่าจะวัด click หรือ submit ตรงไหน
- [ ] รู้ว่า GA4 / Google Tag / Search Console ใช้ทำอะไรโดยรวม

## 8. สิ่งที่ต้องส่ง

ส่งข้อมูลเหล่านี้:

```text
ชื่อโปรเจกต์:
ประเภทเว็บไซต์:
กลุ่มเป้าหมาย:
CTA หลัก:
GitHub Repository:
Vercel URL:
สิ่งที่ทำเสร็จแล้ว:
สิ่งที่จะพัฒนาต่อ:
```

## 9. คำถามสะท้อนผลหลังเรียน

ตอบสั้น ๆ:

1. ส่วนไหนของเว็บที่คุณพอใจที่สุด
2. ส่วนไหนที่ยังต้องปรับ
3. คุณใช้ AI ช่วยตรงไหนบ้าง
4. ครั้งต่อไปคุณจะสั่ง AI ให้ดีขึ้นอย่างไร
5. เว็บนี้จะนำไปใช้ต่ออย่างไร
